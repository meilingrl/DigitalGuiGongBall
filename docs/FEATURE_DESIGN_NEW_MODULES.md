## 一、功能概述

在现有 Hub/Atelier 双循环架构基础上，本轮新增两个 Hub 子模块：

| 模块 | 导航标识 | 核心价值 |
|---|---|---|
| **Explore（探索）** | Hub 左侧导航新增 | 将线下非遗场馆与活动接入平台，打通线上-线下体验链路 |
| **Mall（文创商城）** | Hub 左侧导航新增 | 以积分为纽带，激励用户在知识、创作、活动三个维度持续参与 |

**更新后 Hub 导航结构：**

```
Hub
├── Dashboard（首页）
├── Workshop（创意工坊）
├── Museum（知识库）
├── Social（社区）
├── Explore（探索）          ← 新增
├── Mall（文创商城）          ← 新增
└── Stats（数据）
```

## 二、探索模块（Explore）

### 2.1 信息架构

```
Explore
├── 地图视图（默认）
│   ├── 全国非遗场馆分布标注（Pin）
│   ├── 点击 Pin → 场馆侧边详情面板
│   └── 搜索框（可按城市/类型筛选）
└── 活动日历（Tab 切换）
    ├── 月视图日历
    ├── 点击日期 → 当日活动列表
    └── 点击活动 → 活动详情页
        ├── 封面图 / 标题 / 时间 / 地点
        ├── 活动描述
        ├── "立即预约"按钮
        └── 预约成功模拟弹窗（含签到入口说明）
```

### 2.2 地图交互设计

**技术选型**：高德地图 JavaScript API v2.0（免费 Key，申请链接：https://lbs.amap.com/）

**地图核心交互行为：**

| 交互 | 行为 | 备注 |
|---|---|---|
| 默认加载 | 全国视角，展示所有场馆 Pin | 自定义 Pin 样式（非遗主题图标） |
| 点击 Pin | 右侧弹出场馆详情面板，地图不跳转 | 面板可关闭 |
| 搜索/筛选 | 输入城市名，地图飞行定位 + 高亮对应 Pin | 用高德 `map.setCity()` 实现 |
| 缩放/拖拽 | 原生地图手势，不限制 | 保留高德默认控件 |
| 切换日历 Tab | 地图隐藏，日历视图显示 | 路由级切换，保持地图状态 |

**自定义 Pin 样式方案：**
- 使用高德 `AMap.Marker` + 自定义 `content` HTML
- Pin 颜色：主题色（建议朱砂红 `#C0392B` 或国粹金 `#C9A84C`）
- Pin 图标：可用 SVG 内联（博物馆剪影或非遗纹样）

### 2.5 活动预约与签到流程

```
用户点击"立即预约"
    ↓
模拟预约表单（姓名 / 手机号 / 人数选择）
    ↓
确认预约 → 弹窗：预约成功
    弹窗内容：
    - 活动名称 / 时间 / 地点
    - "活动当天可在此页面签到获取积分（+N 分）"
    - [查看我的预约] 按钮（跳转至 Social 或个人记录）
    ↓
活动当天（模拟）：活动详情页出现"签到"入口按钮
    ↓
点击签到 → 积分发放确认弹窗（+N 分，可在商城查看）
```

**设计原则**：签到入口仅在活动详情页内部出现，**不做全局推送/自动弹窗**，不干扰其他模块体验。

## 三、文创商城（Mall）

### 3.1 信息架构

```
Mall
├── 我的积分卡（顶部横幅）
│   ├── 当前积分显示
│   └── "积分任务"按钮 → 侧边任务面板
│       ├── 任务列表（附完成状态）
│       └── [一键领取]按钮（领取已完成任务积分）
├── 商品列表（主体区域）
│   ├── 分类 Tab（全部 / 文具 / 装饰 / 服饰 / 限定）
│   ├── 商品卡片（封面图 / 名称 / 积分价 / 库存）
│   └── 排序：积分从低到高 / 新品优先
└── 商品详情页
    ├── 图文介绍
    ├── 所需积分 / 当前积分对比
    └── [立即兑换] → 兑换确认弹窗 → 兑换成功页
```

### 3.2 积分体系设计

**核心原则**：积分获取路径对用户在其他模块的操作**无感**，用户仅在主动进入商城、点开任务面板后，才感知并领取积分。其他模块只在完成相应行为时做轻量提示（角标或短暂 Toast）。

#### 3.2.1 积分来源与规则

| 来源 | 触发行为 | 积分奖励 | 可演示方式 |
|---|---|---|---|
| **知识答题（Museum）** | 完成一套非遗知识竞赛 | +30（满分 +50） | 答题完成页显示得分，积分任务标记为"已完成" |
| **创意工坊（Workshop）** | 在创意工坊提交作品 | +40 | 提交表单完成后，积分任务标记为"已完成" |
| **活动签到（Explore）** | 在活动详情页点击签到 | +20~50（随活动设定） | 签到确认弹窗后，积分任务标记为"已完成" |
| **首次访问商城** | 第一次进入 Mall 模块 | +10 | 自动完成，引导用户查看任务面板 |

#### 3.2.2 积分任务面板 UI 规格

点击"积分任务"按钮，右侧滑出任务面板（Drawer 样式）：

```
┌─────────────────────────────┐
│  积分任务               [×] │
├─────────────────────────────┤
│  ✅ 首次访问商城      +10   │
│  ✅ 完成非遗知识竞赛  +50   │
│  ✅ 创意工坊提交作品  +40   │
│  ⬜ 参与活动签到      +30   │
├─────────────────────────────┤
│  待领取积分：  100 分       │
│  [  一键领取  ]             │
└─────────────────────────────┘
```

- 已完成未领取：显示积分数值 + 高亮
- 未完成：置灰，显示提示文字（如"前往 Museum 答题"）
- "一键领取"点击后：播放积分飞入动画，顶部积分数字更新，面板关闭

## 四、知识竞赛模块（Museum 扩展）

### 4.1 题目规格

**一套完整试卷**：10 道题，单选题为主（可含 1~2 道判断题）  
**满分**：100 分（每题 10 分）  
**得分与积分映射**：
- 60 分以下：+20 分（鼓励参与）
- 60~89 分：+30 分
- 90~100 分：+50 分（满分奖励）

### 4.2 题目内容（预设）

| 题号 | 题目 | 选项 | 答案 |
|---|---|---|---|

## 五、创意工坊提交流程（Workshop 扩展）

### 5.1 提交入口


### 5.2 提交表单规格

```
作品标题（必填，限50字）
作品描述（必填，限500字）
作品类型（单选）：
    ○ 纹样创作  ○ 结构设计  ○ 工艺记录  ○ 其他

    鬼工球 / 苏绣 / 蜀绣 / 广绣 / 其他
上传图片（选填，最多3张，仅 UI 展示，不实际上传）
[提交作品]
```

### 5.3 预设示范作品（填充内容）

系统预先展示 2~3 个示范作品，供用户参考：

**示范作品 A**
- 标题：五层鬼工球结构探究
- 描述：本作品尝试在数字环境中还原五层鬼工球的空间嵌套逻辑，通过爆炸视图拆解各层运动关系，并提取外层八瓣莲花纹作为独立装饰元素进行二次创作。
- 类型：结构设计
- 参考项目：鬼工球

**示范作品 B**
- 标题：苏绣配色系统提取
- 描述：以传统苏绣"错针绣"工艺为研究对象，提取十六组经典配色，建立数字配色卡用于现代纹样设计参考。每组配色附有原始绣品图样和色值标注。
- 类型：纹样创作
- 参考项目：苏绣

### 5.4 提交完成后交互

```
点击[提交作品]
    ↓
模拟提交动画（进度条 / 0.5~1s）
    ↓
成功提示 Toast（右上角）：
    "作品已提交！+40 积分已计入，前往商城领取"
    （Toast 3秒后自动消失，不阻断操作）
    ↓
作品出现在 Workshop 展示列表（置顶，标注"我的作品"）
```

## 六、模块交互联动总览

```
Museum（知识竞赛）─────────────────┐
                                    │ 任务完成信号（Pinia store）
Workshop（作品提交）────────────────┤
                                    ↓
Explore（活动签到）─────────────── Mall.积分任务面板
                                    │
                                    ↓（用户主动点击"一键领取"）
                                  积分余额更新 → 商城兑换可用
```

**Pinia Store 设计建议（`usePointsStore`）：**

```typescript
interface PointsStore {
  totalPoints: number        // 当前积分余额
  pendingPoints: number      // 待领取积分
  tasks: {
    id: string
    label: string
    points: number
    completed: boolean       // 行为已触发
    claimed: boolean         // 积分已领取
  }[]
}

// 关键 action
completeTask(taskId: string): void    // 其他模块调用，标记任务完成
claimAll(): void                      // 商城"一键领取"调用
```

## 十、高德地图接入快速指引

1. 前往 https://lbs.amap.com/ 注册账号，创建应用，获取 Web JS Key
2. 在 `index.html` 中引入 SDK：
   ```html
   <script src="https://webapi.amap.com/maps?v=2.0&key=YOUR_KEY"></script>
   ```
3. 在 `ExploreMapView.vue` 中：
   ```typescript
   onMounted(() => {
     const map = new AMap.Map('map-container', {
       zoom: 5,
       center: [108.0, 35.0],  // 全国中心视角
     })
     venues.forEach(v => {
       const marker = new AMap.Marker({
         position: [v.lng, v.lat],
         content: `<div class="venue-pin">...</div>`,
       })
       marker.on('click', () => openVenuePanel(v))
       map.add(marker)
     })
   })
   ```
4. 将 Key 存入 `.env` 文件（`VITE_AMAP_KEY=xxx`），避免硬编码
