# “数字鬼工球”Web高保真原型协作文档（Hub -> Atelier）

## 1. 项目目标与范围

### 1.1 总目标
- 在浏览器中交付可运行的高保真原型，覆盖两大核心模式：
  - `The Hub`（客户端内容中心，2D为主）
  - `The Atelier`（沉浸式工作区，3D交互为主）
- 以“可演示、可讲解、可迭代”为优先，不追求完整生产级功能。

### 1.2 本轮范围（MVP）
- 技术栈：`Vue 3 + Three.js + Tailwind CSS + Pinia`
- 先做 `Hub`，后做 `Atelier`
- 复杂能力以“可信模拟”为主（健康提醒、导出流程、协作状态等）
- 视觉风格：简洁、低干扰、弱科技感（去除冗余HUD元素）

### 1.3 非目标（本轮不做）
- 不做真实多端联机协作（先保留接口与模拟数据）
- 不做完整物理雕刻算法
- 不做完整用户系统、鉴权和后端持久化

## 2. 团队角色建议与职责

> 若人数不足，可由同一人兼任多个角色。

### 2.1 产品/项目负责人（PM）
- 维护需求边界与优先级
- 把控里程碑节奏与验收会议
- 汇总风险与对外汇报材料

### 2.2 前端负责人（Hub）
- 搭建路由、页面结构、状态管理
- 负责 Dashboard/Workshop/Museum/Social/Stats 的界面骨架
- 负责中英双语文案结构接入

### 2.3 3D负责人（Atelier）
- 搭建 Three.js 场景与多层球体程序化生成
- 实现旋转、缩放、层级切换、PIP（微距透视窗）
- 实现风险预警与工具抽屉的交互联动

### 2.4 视觉与交互负责人（UI/UX）
- 维护设计令牌（色彩、间距、层级、字体）
- 统一“去科技感”视觉规范
- 输出关键页面评审稿与交互文案

### 2.5 测试与集成负责人（QA/Integrator）
- 制定验收清单并执行冒烟测试
- 跟踪关键缺陷与回归测试
- 维护演示脚本（答辩流程）

## 5. 验收标准（Definition of Done）

## 5.1 功能验收（必须全部通过）
- F1 可从 Hub 进入 Atelier 并可返回 Hub
- F2 Hub 至少包含：模板卡片、筛选、导航切换

- F5 健康提醒、导出流程、协作状态三项均有可演示结果（允许模拟）
- F6 双语开关可切换，中文为默认语言
- F7 Explore 模块：高德地图加载，四馆 Pin 显示，点击可查看详情
- F8 Explore 模块：活动日历可浏览，活动详情可进入，预约流程可模拟完成
- F9 Mall 模块：商品列表可浏览，积分任务面板可打开，一键领取动效正常
- F10 Mall 模块：兑换流程完整可演示（详情 → 确认 → 成功）
- F11 Museum 竞赛：10道题可完整答题，结果页显示得分与积分
- F12 Workshop 提交：表单可填写提交，提交后 Toast 提示正常
- F13 三条积分路径（答题/提交/签到）与 Mall 任务面板状态正确联动

## 5.2 视觉验收（必须全部通过）

- P3 连续演示 3 分钟不崩溃、不白屏

- C1 目录结构清晰，关键模块有注释说明

## 8. 风险与应对

- 风险R1：3D实现难度超预期  
  - 应对：优先保证“旋转/缩放/层级切换/PIP”，其余功能模拟化

- 风险R2：视觉风格跑偏（又回到重科技风）  
  - 应对：建立UI基线页，先统一组件再扩展页面

- 风险R3：联调冲突多、进度拖延  
  - 应对：模块边界前置、固定每日集成窗口

- 风险R4：演示当天性能不稳定  
  - 应对：准备“性能保底模式”（降低后处理、减少粒子/阴影）

- 風险R5：高德地图 API Key 申请延误  
  - 应对：提前申请；备选方案为使用 Leaflet + OpenStreetMap（免费，无需 Key）临时替代

- 風险R6：积分联动跨模块状态复杂，联调难度高  
  - 应对：usePointsStore 接口前置约定清晰，各模块只调用 completeTask(id) 单一接口，降低耦合

- 風险R7：新功能开发挤压答辩准备时间  
  - 应对：阶段 F/G/H 独立于原 A-E 进度线，新功能 P2 级别验收项允许降级为截图+说明演示

## 9. 最终演示建议流程（5-8分钟，含新模块）

**主线 A：Hub → Atelier 核心体验（3分钟）**
1. 打开 Hub，展示模板筛选与非遗文化入口
2. 选择模板并进入 Atelier
3. 展示 3D 球体交互：旋转、缩放、层级切换
4. 展示 PIP 微距透视窗
5. 触发健康提醒与导出模拟

**主线 B：线上-线下联动体验（2分钟）**
6. 进入 Explore，展示非遗场馆地图，点击场馆 Pin 查看详情
7. 切换至活动日历，进入活动详情，演示预约流程（模拟成功弹窗）

**主线 C：积分体系闭环（2分钟）**
8. 在 Museum 模块进入知识竞赛，完成答题，展示结果与积分提示
9. 进入 Mall，打开积分任务面板，演示一键领取动效
10. 选择商品，完成兑换流程（确认 → 成功页）
