# “数字鬼工球”答辩与报告关键设计要点（HCI + 软件工程方法）

## 1. 课题定位与研究问题

## 1.1 课题定位
- 项目类型：非遗数字化 + 沉浸式交互 + UGC社区平台。
- 设计目标：在不依赖真实象牙材料的前提下，复现“鬼工球”多层镂空雕刻的空间认知挑战与审美体验。

## 1.2 研究问题（可直接写进报告）
- RQ1：如何将高门槛工艺转化为普通用户可参与的数字交互体验？
- RQ2：如何在“社区信息效率”与“沉浸操作专注”之间实现界面解耦？
- RQ3：如何通过实时反馈降低三维微操中的非受迫性失误？
- RQ4：如何让数字原型同时具有文化传播价值与工程可实现性？

---

## 2. 用户研究与人物模型要点

## 2.1 核心用户分层
- 主用户（林锐）：偏挑战驱动，关注精准微操、低失误、协作效率。
- 次用户（陈可）：偏灵感驱动，关注结构拆解、纹样提取、视觉表达。

## 2.2 关键痛点归纳
- 空间认知负担高：多层嵌套结构难以建立稳定心智模型。
- 操作容错率低：深层雕刻中误触代价高、回退成本大。
- 传统展示单向：非遗内容“可看不可用”，缺乏创作闭环。

## 2.3 机会点（设计切入）
- 用“Hub/Atelier双循环架构”解决不同任务心流冲突。
- 用“微距透视窗 + 风险预警”提升复杂三维任务可控性。
- 用“UGC工坊”形成文化传播的再创作机制。

---

## 3. 设计原则（可作为答辩方法论主线）

## 3.1 Principle A：任务导向分层（Task-Layered Design）
- 将“浏览/社交”与“雕刻/专注”分离到两个场景，避免上下文打断。

## 3.2 Principle B：最小干扰反馈（Minimal Intrusion Feedback）
- 使用边缘呼吸光、轻量提示替代中心弹窗，减少注意力抢占。

## 3.3 Principle C：可逆与容错优先（Error-Tolerant Interaction）
- 对高风险动作进行预警和阻断模拟，优先保护用户操作连续性。

## 3.4 Principle D：从文化展示到文化生产（From Display to Creation）
- 不只展示非遗内容，而是提供模板、拆解、再创作路径。

---

## 4. 总体交互架构设计（报告重点图示建议）

## 4.1 双循环模式
- 循环1（Hub）：发现内容 -> 选择模板 -> 进入任务。
- 循环2（Atelier）：沉浸雕刻 -> 提取纹样 -> 回流社区分享。

## 4.2 信息架构（IA）
- Hub：Dashboard / Workshop / Museum / Social / **Explore** / **Mall** / Stats。
  - **Explore（探索）**：线下非遗场馆地图（高德地图 API）+ 活动日历 + 活动预约与签到
  - **Mall（文创商城）**：文创商品兑换 + 积分任务面板（一键领取） + 兑换流程
- Atelier：主视窗（3D对象）+ 抽屉工具 + 微距PIP + 状态反馈边缘层。
- **积分体系（跨模块串联）**：
  - Museum 知识竞赛答题 → 积分奖励
  - Workshop 创意作品提交 → 积分奖励
  - Explore 活动签到 → 积分奖励
  - Mall 积分任务面板一键领取 → 商品兑换

## 4.3 状态流转（State Flow）
- 模板选择状态（Hub）通过 Pinia 传入 Atelier。
- Atelier中的层级、工具、风险状态独立维护，退出后可回流总结。

## 5. 关键交互创新点（建议按“痛点-方案-价值”讲）

## 5.1 微距透视窗（Macro-PIP）
- 痛点：内部层雕刻时“盲雕”严重。
- 方案：双摄像机渲染，同步显示局部放大视图。
- 价值：提升深层结构可见性，降低空间判断错误。

## 5.2 智能防错阻断（Haptic Guard 原型）
- 痛点：误切割导致任务中断、挫败感强。
- 方案：风险区判定 + 边缘预警 + 动作回弹模拟。
- 价值：将“错误代价”前置为“错误预防”。

## 5.3 多层穿梭与协作占位
- 痛点：多人协作时层级冲突与状态不透明。
- 方案：层级独立工作视图 + 队友状态模拟标记。
- 价值：为后续实时协作（Socket）提供稳定交互语义。

## 5.4 全息拆解与纹样提取（演示版）
- 痛点：美学元素难以从复杂结构中直接复用。
- 方案：爆炸视图与提取按钮，导出流程用等待动画模拟。
- 价值：打通“文化体验 -> 设计资产”的转化链路。


## 5.5 线下场馆地图与活动日历（Explore 模块）
- 痛点：平台线上体验孤立，与线下非遗文化活动完全断开。
- 方案：接入高德地图 API，标注全国四大非遗场馆；活动日历展示展览/传习课；模拟预约与签到流程。
- 价値：打通「数字平台 -> 线下文化现场」的联动链路，拓宽非遗传播路径。

## 5.6 积分体系 + 文创商城（Mall 模块）
- 痛点：用户与平台的交互停留在单次体验，缺乏持续参与激励。
- 方案：以「无感积累、主动领取」为原则，用户在知识竞赛/工坊创作/活动签到中自然产生积分，在商城任务面板一键领取，再兑换文创商品。
- 价値：形成「创作/学习/参与 → 积分 → 兑换」的正向激励闭环，强化平台粥性与文化认同。

## 6. 工程化决策与技术合理性（答辩常问）

## 6.1 选型理由
- Vue3：组件化清晰，适合双场景路由与状态组织。
- Pinia：跨页面状态传递简洁，适合模板选择到任务执行流。
- Tailwind：快速构建一致视觉系统，便于迭代审美风格。
- Three.js：直接控制多相机、材质、层级与空间交互。

## 6.2 渐进式实现策略
- 第一阶段：先保证“可演示闭环”（真实交互 + 可信模拟）。
- 第二阶段：逐步替换模拟模块为真实服务（实时协作、导出引擎）。

## 6.3 关键工程权衡
- 先稳定演示链路，再追求算法细节深度。
- 先程序化3D对象，再替换高精模型，降低外部素材依赖风险。

## 7. 可用性评估方案（课程方法分关键加分项）

## 7.1 评估目标
- 验证用户是否能高效完成“选模板 -> 进入雕刻 -> 完成一次层级操作 -> 导出”流程。
- 验证关键反馈机制是否降低误操作与认知负担。

## 7.2 评估方法（建议混合）
- 任务驱动可用性测试（Task-based Usability Test）
- 思维大声说出法（Think-Aloud）
- SUS量表（System Usability Scale）
- NASA-TLX（可选，评估认知负荷）

## 7.3 核心指标（可量化）
- TCT：任务完成时间（Task Completion Time）
- SR：任务成功率（Success Rate）
- ER：误操作率（Error Rate）
- RL：恢复时间（Recovery Latency）
- SUS均分（目标可设 >= 75）

## 7.4 A/B验证建议（可写未来工作）
- A组：无PIP；B组：有PIP。对比深层任务时间与错误率。
- A组：弹窗预警；B组：边缘呼吸预警。对比打断感与完成率。

## 8. 非功能与伦理合规（报告升维）

## 8.1 文化与伦理价值
- 完全数字化替代现实象牙工艺练习，规避生态与法律风险。
- 非遗由“单向观看”升级为“参与式活态传承”。

## 8.2 健康与安全
- 长时操作引入健康提醒，降低视觉疲劳与姿态负担。
- 风险反馈机制减少高压微操带来的心理挫败。

## 8.3 可扩展性
- 架构上预留协作通信接口与素材管理扩展点。
- 可向教育场景（工艺教学）与设计场景（纹样资产库）延展。

## 9. 答辩结构建议（5-8分钟）

1. **问题与价值**：为什么是“数字鬼工球”，为什么必须是沉浸交互。  
2. **用户与痛点**：两类用户与冲突需求。  
3. **设计方法**：双循环架构 + 四条设计原则。  
4. **核心演示**：Hub到Atelier、PIP、風险预警、模拟导出；**Explore**地图与活动预约；**Mall**积分领取与兑换。  
5. **评估结果/计划**：指标、测试方法、当前结果或预期。  
6. **总结与展望**：从高保真原型走向可落地系统。

## 10. 报告可直接复用的小结段落（示例）

本项目采用“社区内容中心（Hub）+沉浸工作区（Atelier）”的双循环交互架构，将信息浏览与高专注创作过程进行结构化解耦。通过微距透视窗、边缘化風险反馈与多层级空间操作机制，系统在不引入复杂学习门槛的前提下提升了三维微操任务的可见性、可控性与容错性。在此基础上，项目进一步扩展了线下联动模块（Explore）与激励闭环模块（Mall）：Explore 通过高德地图 API 展示全国非遗场馆分布并联接线下活动预约；Mall 以「无感积累、主动领取」为设计原则，将知识竞赛、工坊创作、活动参与三条路径的用户行为转化为可兑换的文创积分，形成平台参与的正向激励闭环。工程实现上，项目基于 Vue3、Pinia、Tailwind、Three.js 与高德地图 API 构建，以「可演示闭环优先、复杂能力渐进实装」为策略，在课程周期内居照高保真交互表现与软件工程可执行性。
