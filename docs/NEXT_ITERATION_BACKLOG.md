## 零、需求对齐（已确认，2026-04-23）


| 代号               | 结论                                                                                                          | 实现要点                                                                                                                        |
| ---------------- | ----------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| **A（D-4 地图语言）**  | 问题界定：应用切到**英文模式**时，**地图仍显示中文**，与语言模式不一致。                                                                    | **目标**：中英切换时，地图侧展示语言应随 `app.locale` 切换（例如高德 `lang` / 图层与自定义标注文案同步）。若个别控件受地图服务商限制无法切换，再在 UI 上少量说明；**不以**「英文模式永远中文底图」为终态。     |
| **B（F-8 签到与范围）** | 需要**场馆半径**（地理围栏）设计；**不使用真实设备定位**；除「当前坐标来源」为模拟外，**其余流程按真实产品设计**（场馆经纬度、半径、范围内外判定、提示文案、签到按钮状态等）。               | 采用**模拟用户坐标**（如演示开关或单次「假设我在场馆内」）参与距离计算；**不调用**浏览器 Geolocation 作为签到依据（除非你方后续改需求）。半径与场馆数据与 `FEATURE_DESIGN_NEW_MODULES` 保持一致性。 |
| **C（F-4 协作聊天）**  | **早期版本不持久化**；会话内**能发送、能展示**即可。                                                                              | 内存态消息列表 + Mock 好友/会话；刷新页面清空可接受。后续若加持久化再开任务。                                                                                 |
| **D（F-10 内容共建）** | 长期希望有**审核流**；当前**无管理员路由与专属界面**。本阶段：先做**共建提交界面**，提交成功后弹出 **待审核** 等明确提示（Modal/Toast 文案与 `content.ts` 双语一致即可）。 | 不实现审核后台；不新增管理员路由，除非单独排期。                                                                                                    |
| **E（E-1 资产标准化）** | **统一标准化方案待团队讨论**；**未经讨论不得擅自**对仓库内资源做批量重命名、移动路径或改 import。                                                    | 初稿文档中若曾举例某子目录，仅作说明用，**以你方仓库实际为准**。讨论产出约定（目录、命名、格式表）后再动工程文件。                                                                 |

## 一、总体评估结论


| 维度    | 结论                                                                                                  |
| ----- | --------------------------------------------------------------------------------------------------- |
| 与现有定位 | 与 `TEAM_WORKPLAN` 中「高保真原型、可信模拟、非完整后端」一致；多数「功能完善」可通过 **Mock 数据 + 本地状态** 落地，少数需明确是否引入真实服务。            |
| 建议节奏  | **P0** 视觉与文案一致性（登录、博物馆顶栏、地图语言、预览提示）→ **P1** 工坊/博物馆/探索闭环增强 → **P2** 社交协作与预约签到 → **P3** 资产规范与「研究型」能力。 |
| 主要风险  | **地图双语**（第三方底图与自研文案不同步）、**签到与模拟定位**（产品叙事 vs 合规表述）、**协作 Ghost Hand**（无后端时的同步策略）。                     |
